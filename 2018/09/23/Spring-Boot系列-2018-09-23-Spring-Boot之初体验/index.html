Spring Boot之初体验

felixu | 发表于 2018-09-23 | 分类于 Spring Boot | | 阅读次数
Spring Boot是什么，是很多新手初接触Spring Boot的第一个疑问可能是Spring Boot是什么，从我的理解上，Boot Spring便是Spring Boot最好的解释，做Java的应该都接触过Spring，也应该都知道Spring虽然好用，但是哪怕是搭建一个什么功能都没有的空架子也需要花费一番功夫来配置和启动，而Spring Boot便是在Spring的基础上又包了一层来帮助我们更便捷更轻松的启动一个Spring应用。

Spring Boot简介

不做特殊说明，所有Spring Boot的案例均会使用Spring Boot2.x版本

Spring Boot是什么

在刚刚接触Spring Boot的时候，总会看到有一些人拿Spring MVC来和Spring Boot来做比较的，其实我是不太明白这样比较的意义在哪里的(应该是我比较菜)。Spring Boot其实就是一层壳，包裹着Spring的一层壳。有人会拿Spring MVC和Spring来比较吗？Spring官网对Spring Boot的解释中就可以看出Spring Boot的作用

Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can “just run”.

We take an opinionated view of the Spring platform and third-party libraries so you can get started with minimum fuss. Most Spring Boot applications need very little Spring configuration.

Spring Boot的特性

Spring Boot的特点是易学难精，那么为啥会是这样呢，原因在于Spring Boot在使用上，他简化了Spring的配置，使我们可以快速搭建一个Spring应用，反而让我们越来越忽略Spring本来的样子，然而，真正要理解Spring Boot的原理其实还是要深究Spring，从Spring3.1开始，我们不使用Spring Boot其实也是可以做到Spring Boot的自动装配等功能。咦，扯远了，在看Spring Boot的特性之前，先了解几个概念：

应用的分类

功能性应用：与业务相关
非功能性应用： 安全、性能、监控等

外部配置

启动参数、配置文件、环境变量

外部应用

Servlet应用、Spring Web MVC、Spring Web Flux、WebSocket、WebService

Spring Boot特性

官网给出了如下的特性：

  • Create stand-alone Spring applications创建独立的Spring应用
  • Embed Tomcat, Jetty or Undertow directly (no need to deploy WAR files)嵌入式的Tomcat，Jetty，Undertow不需要外部依赖
  • Provide opinionated 'starter' dependencies to simplify your build configuration提供固定的stater依赖来简化配置
  • Automatically configure Spring and 3rd party libraries whenever possible自动配置Spring和第三方库
  • Provide production-ready features such as metrics, health checks and externalized configuration提供为生产而准备的特性，比如指标，健康检查和外部配置
  • Absolutely no code generation and no requirement for XML configuration绝对没有代码生成和XML配置要求

了解更多Spring Boot相关信息。
